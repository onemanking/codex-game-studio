# Upgrading Codex Game Studio

This guide is for projects that were cloned from this template and later want to
pull in template updates.

## Recommended Strategy

Add the template as a Git remote in your game project:

```powershell
git remote add template <your-codex-game-studio-repo-url>
git fetch template main
git merge template/main --allow-unrelated-histories
```

Resolve conflicts by keeping your game-specific content while accepting template
improvements to `.codex/skills`, `.codex/agents`, scripts, docs, and inventories.

## Files To Merge Carefully

- `.codex/docs/technical-preferences.md`
- `.codex/docs/workflow-catalog.yaml`
- `.codex/skills/*` when you have customized skills for a project.
- `.codex/agents/*` when you have customized agent behavior.
- `AGENTS.md` if your project has added local operating rules.

## Files Usually Safe To Accept From Template

- `scripts/*.ps1`
- `sources/upstream-skill-inventory.*`
- Browser-library skill directories regenerated by `scripts/sync-upstream-skills.ps1`
- Validation improvements in `scripts/validate-skills.ps1`

## After Upgrade

Run:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\sync-codex-workspace.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\validate-skills.ps1
```

Then inspect `git diff` and verify any game-specific runtime behavior affected by
the merge.
